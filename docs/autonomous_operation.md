# Autonomous Operation

민오 운영 원칙은 `관여 0%, 앱에서는 관전만`입니다. 따라서 시스템은 사람이 승인하거나 판단해주는 구조가 아니라, 스스로 실행하고 스스로 멈추는 구조여야 합니다.

## 원칙

- 앱은 read-only입니다.
- 수동 진입은 없습니다.
- 주문 승인은 없습니다.
- 포지션 변경은 엔진만 수행합니다.
- 운영자는 코드/설정 배포로만 개입합니다.
- 위험 상황에서는 사람이 누르는 버튼이 아니라 자동 리스크 제한이 먼저 작동합니다.

## 실행 구조

```text
TradingView Alert
-> Cloudflare Worker
-> alert validation / dedupe / stale check
-> Queue
-> Execution Engine
-> State Store
-> Binance
-> Logs / Metrics
-> Flutter Observer App
```

## 자동 정지 조건

필수 자동 정지 조건입니다.

- 일 손실 -2%: 신규 진입 금지
- 주 손실 -5%: 포지션 50% 축소
- 월 손실 -10%: 전체 청산 후 거래 중단
- CHAOTIC: 신규 진입 금지, 기존 포지션 50% 이상 축소
- API 오류 연속 발생: 신규 주문 중단
- 포지션 조회 실패: 신규 주문 중단
- 주문 체결 상태 불명: 신규 주문 중단
- TradingView alert 지연: 해당 alert 폐기
- 중복 alert: 폐기
- 레짐 신호 충돌: RANGE 처리

## 자동 복구 조건

자동 복구는 보수적으로 합니다.

- 쿨다운 종료
- 계좌/포지션 조회 정상
- 미체결 주문 상태 확인 완료
- 레짐 재확인
- 첫 재개 노출은 평소 목표보다 작게 시작

월 손실 제한 이후에는 자동 재개하지 않는 것을 기본값으로 둡니다. 월 손실은 시스템 가정이 틀렸을 가능성이 크기 때문에 코드/설정 점검 후 재개하는 편이 안전합니다.

## 필요한 저장 상태

재시작해도 이어서 판단할 수 있어야 합니다.

- 현재 모드
- 마지막 레짐과 바이어스
- 레짐 히스토리
- 마지막 처리한 `signal_id`
- 쿨다운 종료 시각
- 마지막 리밸런싱 시각
- 마지막 지갑 잔고
- 입금 후 대기 중인 리밸런싱 시각
- 일/주/월 기준 equity
- 주문 계획과 실행 결과

## 앱에서 볼 지표

- 현재 총자산
- 현재 노출과 목표 노출
- 레버리지 사용률
- 레짐 점수
- 세부 바이어스
- 현재 포지션
- 목표 포지션
- 최근 주문
- 자동 정지 여부
- 쿨다운 남은 시간
- 최근 alert
- 최근 오류

## 실거래 전 통과 기준

- paper mode 30일 이상
- 롱/숏 직접 전환 0건
- 중복 alert 실행 0건
- stale alert 실행 0건
- 포지션 불일치 미해결 0건
- 자동 정지 조건 작동 확인
- 재시작 후 상태 복구 확인
- Flutter 앱 없이도 엔진 단독 정상 운영 확인

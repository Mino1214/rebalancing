# 강화 테스트 방향

이 시스템은 수익을 맞히는 테스트보다, 위험한 상태에서 잘 멈추는지 검증하는 테스트가 먼저입니다.

## 1. 단위 테스트

현재 있는 테스트를 더 촘촘히 만듭니다.

- 레짐 점수
  - BTC만 상승
  - BTC + TOTAL 상승, 알트 약세
  - BTC/TOTAL/TOTAL2/TOTAL3 동반 하락
  - BTC.D 상승으로 알트 약세 가산
  - CHAOTIC이 점수보다 우선하는지

- 전환 가드
  - `LONG -> SHORT` 직접 전환 금지
  - `SHORT -> LONG` 직접 전환 금지
  - 중립 12시간 미만이면 대기
  - 쿨다운 중에는 신규 진입 금지

- 포트폴리오
  - 상위 도미넌스 10개만 선택
  - USDT, USDC, DAI 등 스테이블코인 제외
  - `BTC_ONLY_BULL`은 BTC/ETH 중심
  - `BROAD_BULL`은 상위 도미넌스 10개
  - `ALT_WEAK_BEAR`은 약한 알트 숏 비중 증가

- 주문 계획
  - 축소 주문은 reduce-only
  - 큰 주문은 분할
  - 반대 포지션은 먼저 닫고 같은 사이클에 뒤집지 않음

- 리스크
  - 일 -2% 신규 진입 금지
  - 주 -5% 50% 축소
  - 월 -10% 전체 청산 후 정지

## 2. 시나리오 테스트

캔들 하나가 아니라 시간 흐름을 테스트합니다.

```text
BROAD_BULL 3캔들 -> LONG
BULL 약화 -> 축소
RANGE -> 중립
BEAR 3캔들 -> SHORT
CHAOTIC -> 50% 축소 + 24시간 정지
```

그리고 입금 시나리오도 따로 봅니다.

```text
USDT 입금
-> 즉시 진입 없음
-> 다음 4시간봉 마감
-> 레짐 재확인
-> 부족분만 추가
```

## 3. 백테스트

백테스트는 최소 3단계로 나눕니다.

1. 레짐 판정 백테스트
   - 실제 주문 없이 레짐만 기록합니다.
   - BULL/BEAR/RANGE/CHAOTIC 구간이 직관적으로 맞는지 봅니다.

2. 포트폴리오 백테스트
   - 목표 포지션과 리밸런싱만 시뮬레이션합니다.
   - 수수료, 펀딩비, 슬리피지를 보수적으로 넣습니다.

3. 실행 백테스트
   - 지정가 미체결, 분할 주문, reduce-only 실패, 최소 주문 금액을 반영합니다.

## 4. 스트레스 테스트

꼭 넣어야 할 케이스입니다.

- BTC 4시간 -7% 급락
- 급락 후 4시간 +8% 반등
- 펀딩비 과열
- 거래량 3배 폭증
- API 지연/실패
- 일부 주문 체결, 일부 주문 실패
- 포지션 데이터와 주문 데이터 불일치
- 입금 직후 레짐 반전

## 5. 페이퍼 트레이딩 판정 기준

실거래 전 최소 기준입니다.

- 30일 이상 무중단 실행
- 주문 계획과 체결 시뮬레이션 불일치 0건
- 월 손실 제한 작동 확인
- CHAOTIC 정지 작동 확인
- 롱/숏 직접 전환 0건
- 스테이블코인 편입 0건
- 상위 도미넌스 10개 선정 로그 정상
